Python Modules and Packages

Published: November 2025 Category: Python

Introduction

When your code grows beyond a single file, Python provides two tools to organize it effectively: modules and packages. A module is a single Python file, while a package is a directory containing multiple modules.

In this tutorial, we’ll break down how these work, what __init__.py and __pycache__ do, and how imports really function in Python.

What is a Module?

A module is just a Python file that contains related functions, classes, or variables. You can import a module from another file and reuse its code.

text 
MyLibrary/
│
├── main.py
├── calc.py
└── greet.py

calc.py

python 
def add(a, b):
    return a + b

greet.py

python 
def say_hello():
    return "Hello!"

main.py

python 
from calc import add
from greet import say_hello

print(say_hello())
print(add(5, 6))

Running python main.py imports calc.py and greet.py as modules and executes their code.

What is a Package?

A package is a directory that groups together related modules. It usually contains a special file named __init__.py which marks the folder as a package.

text 
MyLibrary/
│
├── main.py
└── utils/
    ├── __init__.py
    ├── calc.py
    └── greet.py

__init__.py

python 
print("Code imported successfully")

main.py

python 
from utils.calc import add
from utils.greet import say_hello

print(say_hello())
print(add(5, 6))

When you run this, Python first executes __init__.py. This confirms that utils is a package and allows its modules to be imported.

Why Use __init__.py?

It serves two main purposes:

  • Package Initialization: Runs setup code when the package is imported.
  • Cleaner Imports: Lets you simplify imports by exposing specific functions.

Example:

python 
# utils/__init__.py
from utils.calc import add
from utils.greet import say_hello

main.py

python 
from utils import add, say_hello

print(add(5, 6))
print(say_hello())

What is __pycache__?

When Python imports a module, it compiles the source file (.py) into bytecode (.pyc) to make imports faster next time. These compiled files are stored automatically in the __pycache__ directory.

You can safely ignore or delete it — Python will recreate it as needed.

Absolute vs Relative Imports

Absolute imports: use the full path from the project root.

python 
from utils.calc import add

Relative imports: use dots to refer to the current or parent package.

python 
from .calc import add
from ..helpers import util_function

Nested Packages

You can organize complex projects using nested packages.

text 
MyLibrary/
│
├── main.py
└── utils/
    ├── __init__.py
    ├── m_util/
    │   ├── __init__.py
    │   └── math_util.py
    └── s_util/
        ├── __init__.py
        └── string_util.py

main.py

python 
from utils.m_util.math_util import add_numbers
from utils.s_util.string_util import reverse_string

Summary

  • Module: a single .py file.
  • Package: a directory containing modules and __init__.py.
  • __init__.py: initializes and organizes your package.
  • __pycache__: stores compiled bytecode for faster imports.
  • Absolute imports: use full paths.
  • Relative imports: are useful within large packages.
